# ADC pinout contract (v1)

This document defines the **signal-level contract** between:
- the IP RTL (`chip-inventory`) and
- the submission harness / top-level wrapper (`home-inventory-chip-openmpw`).

Goal: prevent last-minute pad/port drift by keeping one canonical list of ADC-facing pins and their expected direction/behavior.

Normative references:
- ADC part: TI **ADS131M08** (`decisions/008-adc-part-selection.md`)
- ADC streaming contract: `docs/ADC_STREAM_CONTRACT.md`
- ADC clocking plan: `docs/ADC_CLOCKING_PLAN.md`

---

## 1) Canonical signal list (RTL-facing)

These are the **logical** signals the RTL expects to connect to the ADC.

### SPI interface
- `adc_sclk` (output): SPI clock generated by the chip.
- `adc_cs_n` (output, active-low): chip-select.
- `adc_mosi` (output): controller → ADC.
- `adc_miso` (input): ADC → controller.

### Data-ready / framing
- `adc_drdy_n` (input, active-low): asserted by ADC when a new frame is ready.
  - If the harness does not expose DRDY in v1, document the alternative (polling / timed capture) explicitly.

### Master clock
- `adc_clkin` (input): ADC master clock (CLKIN).
  - **This is a board/harness responsibility** unless we explicitly drive it from a chip clock output.
  - Frequency/quality requirements live in `docs/ADC_CLOCKING_PLAN.md`.

### Reset / power-down (optional but strongly preferred)
- `adc_reset_n` (output, active-low): optional hard reset.
- `adc_pwdn_n` (output, active-low): optional power-down control.

If `adc_reset_n` / `adc_pwdn_n` are not available on the harness, the wrapper must tie them to the safe state and the limitation must be recorded in `docs/ADC_FW_INIT_SEQUENCE.md`.

## 2) Harness mapping (must be filled before tapeout)

The harness currently has an **opt-in GPIO routing block** in:
- `home-inventory-chip-openmpw/verilog/rtl/home_inventory_user_project.v`
  - enabled by compile-time define: `HOMEINV_ENABLE_ADC_GPIO`
  - uses parameterized indices (currently placeholders)

For each logical ADC signal, record **exactly one** of:
- Caravel user GPIO (`io[<n>]`)
- explicit net name in the wrapper (`user_project_wrapper`/`home_inventory_user_project`)
- "not connected" (with reason)

### Current harness placeholders (DO NOT TAPEOUT AS-IS)

As of today, the harness uses these parameter defaults when `HOMEINV_ENABLE_ADC_GPIO` is enabled:

```text
adc_sclk     -> io[0]   (ADC_SCLK_IO)
adc_cs_n     -> io[1]   (ADC_CSN_IO)
adc_mosi     -> io[2]   (ADC_MOSI_IO)
adc_miso     -> io[3]   (ADC_MISO_IO)      [input]
adc_drdy_n   -> io[4]   (ADC_DRDYN_IO)     [input]
adc_reset_n  -> io[5]   (ADC_RSTN_IO)
adc_clkin    -> not connected (must be provided by board/harness)
adc_pwdn_n   -> not connected (optional; add when pads allow)
```

Notes:
- `adc_reset_n` is currently **driven high** in the harness (`assign adc_rst_n = 1'b1;`).
  - i.e. the pad is reserved, but the IP does not yet own reset sequencing.
- `adc_drdy_n` is currently only routed to a harness-local wire (it is **not** yet consumed by the IP ingest RTL).
- When `HOMEINV_ENABLE_ADC_GPIO` is *not* enabled, the wrapper ties `adc_miso=0` and `adc_drdy_n=1`.

## 3) Build-flag behavior (port-list stability)

In `chip-inventory`, the Wishbone block conditionally exposes ADC pins only when:
- `USE_REAL_ADC_INGEST` is defined

Contract:
- Harness must compile-clean in both modes:
  - default (stub SNAPSHOT path)
  - `-DUSE_REAL_ADC_INGEST` (real pin exposure)

Fast checks:
- IP repo: `bash ops/rtl_compile_check.sh`
- Harness repo: `make rtl-compile-check` and `make rtl-compile-check-real-adc`

## 4) Tapeout gates (pinout-related)

Before tagging a tapeout candidate:
1) This file’s mapping section is fully filled (no `TBD`).
2) `docs/ADC_CLOCKING_PLAN.md` names the CLKIN source (oscillator vs chip clock) + frequency.
3) Harness compile-check passes with `USE_REAL_ADC_INGEST` enabled.
4) A bring-up note exists for each missing optional pin (reset/pwdn/drdy).
