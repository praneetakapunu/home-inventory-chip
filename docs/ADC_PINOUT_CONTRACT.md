# ADC pinout contract (v1)

This document defines the **signal-level contract** between:
- the IP RTL (`chip-inventory`) and
- the submission harness / top-level wrapper (`home-inventory-chip-openmpw`).

Goal: prevent last-minute pad/port drift by keeping one canonical list of ADC-facing pins, their direction/polarity, and which ones are **actually required by the current RTL**.

Normative references:
- ADC part: TI **ADS131M08** (`decisions/008-adc-part-selection.md`)
- ADC streaming contract: `docs/ADC_STREAM_CONTRACT.md`
- ADC clocking plan: `docs/ADC_CLOCKING_PLAN.md`

---

## 1) Canonical signal list (logical)

These are the **logical** signals we may connect to the ADC.

### A) Required by current RTL (today)
The current v1 “real ADC ingest” RTL only exposes the SPI pins:
- `adc_sclk` (output): SPI clock generated by the chip.
- `adc_cs_n` (output, active-low): chip-select.
- `adc_mosi` (output): controller → ADC.
- `adc_miso` (input): ADC → controller.

This matches the port list of:
- `rtl/home_inventory_wb.v` (when `USE_REAL_ADC_INGEST` is enabled)
- `rtl/home_inventory_top.v`
- `rtl/adc/adc_streaming_ingest.v`

### B) Strongly preferred for bring-up (not yet in RTL top-level)
These pins are **expected to exist on the harness/board** for a robust bring-up, but the current RTL top-level does not yet consume/expose them:
- `adc_drdy_n` (input, active-low): asserted by ADC when a new frame is ready.
- `adc_rst_n` (output, active-low): hard reset to ADC (recommended).
- `adc_clkin` (input): ADC master clock (CLKIN). (Source/frequency requirements live in `docs/ADC_CLOCKING_PLAN.md`.)
- `adc_pwdn_n` (output, active-low): optional power-down control.

If the harness provides these today, we should still **lock** the mapping here (so we can wire them into RTL later without guessing).

## 2) Harness mapping (must be filled before tapeout)

For each logical ADC signal, record **exactly one** of:
- Caravel user GPIO (`io[<n>]`)
- explicit net name in the wrapper (`user_project_wrapper` / `home_inventory_user_project`)
- "not connected" (with reason)

### Current harness placeholders (DO NOT TAPEOUT AS-IS)

As of today, the harness uses placeholder indices when its ADC GPIO routing block is enabled.

Record the final mapping here once confirmed:

```text
adc_sclk    -> io[?]   (TBD)
adc_cs_n    -> io[?]   (TBD)
adc_mosi    -> io[?]   (TBD)
adc_miso    -> io[?]   (TBD)   [input]
adc_drdy_n  -> io[?]   (TBD)   [input]
adc_rst_n   -> io[?]   (TBD)
adc_clkin   -> ??? (oscillator or SoC clock-out; see docs/ADC_CLOCKING_PLAN.md)
adc_pwdn_n  -> (optional)
```

Notes:
- If the harness currently names the reset net `adc_rst_n` (not `adc_reset_n`), **this document standardizes on `adc_rst_n`**.
- DRDY polarity: treat **assert = low**. If the harness uses `adc_drdy` naming (without `_n`), it must be explicitly documented as active-low or active-high.

## 3) Build-flag behavior (port-list stability)

In `chip-inventory`, the Wishbone block conditionally exposes the SPI pins only when:
- `USE_REAL_ADC_INGEST` is defined

Contract:
- Harness must compile-clean in both modes:
  - default (stub SNAPSHOT path)
  - `-DUSE_REAL_ADC_INGEST` (real SPI pin exposure)

Fast checks:
- IP repo: `bash ops/rtl_compile_check.sh`
- Harness repo: `make rtl-compile-check` and `make rtl-compile-check-real-adc`
- Harness polarity/name audit (grep-only):
  - `tools/harness_adc_drdy_audit.sh ../home-inventory-chip-openmpw`

---

## 4) Tapeout gates (pinout-related)

Before tagging a tapeout candidate:
1) This file’s mapping section is fully filled (no `TBD`).
2) `docs/ADC_CLOCKING_PLAN.md` names the CLKIN source (oscillator vs chip clock) + frequency.
3) Harness compile-check passes with `USE_REAL_ADC_INGEST` enabled.
4) For each missing bring-up-preferred pin (`adc_drdy_n`, `adc_rst_n`, `adc_pwdn_n`), there is an explicit bring-up note describing the limitation and mitigation.
